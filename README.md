# DuPharma
A centralized, .NET based enterprise application to manage pharmaceutical inventory, prescriptions, sales and reporting for community and hospital pharmacies.
## 1. Project Name
DuPharma
## 2. Problem Statement
Many local pharmacies still depend on manual or semi‑manual methods to manage inventory and prescriptions. This results in:
- Frequent stockouts and overstocking due to poor tracking.
- Medicines expiring unnoticed, leading to financial losses.
- Difficulty maintaining accurate prescription history for customers.
- Human errors in sales records and reporting.
- Lack of centralized visibility for admins and managers across branches.
- Time‑consuming report generation and poor decision-making tools.
These inefficiencies reduce productivity, lower service quality, and create compliance risks.
## 3. Objective
To develop a centralized, user‑role‑based pharmacy management system that:
- Tracks real‑time medicine stock levels and expiry dates.
- Allows pharmacists to record sales and manage customer prescriptions.
- Enables managers to supervise stock, handle returns, and generate reports.
- Provides admins with full system control, user management, and report dashboards.
- Ensures security, accuracy, and automation of daily pharmacy operations.
- Generates analytics such as most sold medicines, daily reports, and alerts.
